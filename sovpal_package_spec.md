# `sovpal`: Soviet & Industrial Color Palettes for R
## Package Build Specification for Claude Code

---

## 1. Overview & Goal

Build a complete, installable R package called `sovpal` that provides color palettes derived from Soviet military, industrial, institutional, and safety signage color systems. The package should follow the design patterns of `{MetBrewer}` and `{wesanderson}`, with full `{ggplot2}` integration via scale functions.

**Core design philosophy:** All hex values are historically grounded and must not be modified to suit aesthetic preferences. Usability on white backgrounds is achieved through *curation and subsetting*, not by altering the canonical colors. The package is simultaneously a data visualization tool and a faithful record of Soviet color systems.

The package should be ready for GitHub publication (not CRAN initially). It must pass `R CMD check` with 0 errors, 0 warnings.

## 4. Palette Data (`R/palettes.R`)

### 4.1 Core palette list

Define a single internal named list called `.sovpal_palettes`. Each entry is a named character vector of hex codes with a `type` attribute (`"qualitative"`, `"sequential"`, or `"diverging"`). **No hex value in this list may be modified for aesthetic reasons.** Every color must be traceable to a historical source documented in the verification table in section 13.

### 4.2 Qualitative Palettes

#### `gost14202`
The complete 8-color Soviet industrial pipeline identification system (GOST 14202-69, adopted 1969, still mandatory in post-Soviet industrial facilities). Colors reference the standard's physical card index samples: green №343-344, red №10-11, blue №423-424, yellow №205-206, orange №101-102, violet №505-506, brown №647-648, grey №894-895.

```r
gost14202 = c(
  water  = "#2E7D32",  # green  — water mains (питьевая вода)
  fire   = "#C62828",  # red    — fire suppression / superheated steam
  air    = "#5B8DB8",  # blue   — compressed air (lightened per spec: blue + white)
  gas    = "#F9A825",  # yellow — flammable gases, acids
  toxic  = "#E64A19",  # orange — toxic / corrosive substances
  alkali = "#8E6EAF",  # violet — alkalis (lilac-violet per spec)
  fuel   = "#5D4037",  # brown  — combustibles, oils, fuel
  other  = "#78909C"   # grey   — condensate, coolant, misc fluids
)
```

#### `soviet_military`
Soviet WWII and Cold War era vehicle and aircraft colors. Sources: Vallejo 70.609 / RAL6025 for 4BO; AK Real Colors RC206 for cockpit turquoise; FS30117 equivalent for 6K; FS23578 equivalent for 7K.

```r
soviet_military = c(
  green_4bo  = "#51653F",  # 4BO Protective Green exterior
  turquoise  = "#5B8F8C",  # cockpit/interior turquoise
  primer     = "#C45830",  # red lead primer (свинцовый сурик)
  soviet_red = "#CD2500",  # flag and insignia red
  brown_6k   = "#3D2B1F",  # Dark Brown 6K camouflage
  sand_7k    = "#C9A96E"   # Yellow Earth 7K camouflage
)
```

#### `constructivist`
Soviet Constructivist graphic design (Rodchenko, Lissitzky, early Pravda). Cream (#E8DFC8) is the color of aged Soviet newsprint — historically essential. On white backgrounds, use the 4-color subset (see section 4.4) which omits cream rather than replacing it with a non-historical color.

```r
constructivist = c(
  red    = "#CD2500",  # Soviet red
  black  = "#1A1A1A",  # near-black
  yellow = "#F9A825",  # signal yellow
  cream  = "#E8DFC8",  # aged newsprint / propaganda poster ground
  blue   = "#1565C0"   # deep propaganda blue
)
```

#### `gost_signal`
The four mandatory signal colors from GOST 12.4.026 (Soviet/Russian workplace safety signage standard). Semantically encoded: red = immediate danger/prohibition; yellow = caution/warning zones; green = safe condition/evacuation; blue = mandatory action/PPE.

```r
gost_signal = c(
  danger    = "#D32F2F",  # red    — immediate danger, prohibition
  caution   = "#FDD835",  # yellow — warning, hazard indication
  safe      = "#388E3C",  # green  — safe condition, evacuation routes
  mandatory = "#1565C0"   # blue   — mandatory action, PPE required
)
```

### 4.3 Sequential Palettes

Each sequential palette defines 4–5 anchor colors from light (low) to dark (high). The `sovpal()` function interpolates when `n` exceeds the number of defined stops.

#### `pripyat`
A desaturated green sequence evoking Pripyat's abandoned interiors: aged cream walls fading through institutional green to graphite. Stop 1 (cream) is historically accurate but near-white; use the `n = 2` subset as the light anchor when plotting on white (see section 4.4).

```r
pripyat = c(
  "#E8DFC8",  # cream wall (Soviet institutional interior)
  "#C5C9A8",  # faded institutional yellow-green
  "#7B9E7B",  # institutional green
  "#51653F",  # 4BO green
  "#3A3A38"   # reactor graphite
)
```

#### `reactor`
A near-neutral blue-grey sequence evoking shielding panels and Soviet concrete. Stop 1 (anti-flash white #F2F3F4) is the documented color of Soviet nuclear bomber undersides. Near-white on white backgrounds; use `from = 2` parameter (see section 5.1) to start from stop 2 when needed.

```r
reactor = c(
  "#F2F3F4",  # anti-flash white (nuclear bomber undersides)
  "#C5C8CA",  # light lead grey
  "#78909C",  # pipe grey / coolant
  "#6B6B6E",  # lead shield grey
  "#3A3A38"   # reactor graphite
)
```

#### `steppe`
The three Soviet camouflage field colors in terrain order. All stops have adequate contrast on white; no subsetting needed.

```r
steppe = c(
  "#C9A96E",  # Yellow Earth 7K (sand/soil)
  "#7B9E7B",  # field/institutional green
  "#51653F",  # 4BO Protective Green
  "#3D2B1F"   # Dark Brown 6K
)
```

### 4.4 Diverging Palettes

#### `hazard`
Semantically grounded: low = safe (GOST pipe green), mid = neutral (GOST pipe grey), high = danger (GOST pipe red). All stops from gost14202. Suitable for risk scores, hazard ratios, p-values, survival data. No white-background issues; all stops adequately contrast.

```r
hazard = c(
  "#2E7D32",  # gost14202 water green (safe)
  "#7B9E7B",  # muted green transition
  "#78909C",  # gost14202 pipe grey (neutral)
  "#C62828",  # gost14202 fire red (danger)
  "#8B0000"   # deep danger
)
```

#### `coldwar`
Metro cobalt → institutional cream → Soviet red. The cream midpoint (#E8DFC8) is the same institutional wall color as in `pripyat` — a physically grounded neutral. On white backgrounds, the cream midpoint has reduced contrast; use `from = 2, to = 4` to drop the outer stops and tighten the range, or acknowledge the reduced visual anchor (documented in section 4.5). Do not replace cream with a non-historical color.

```r
coldwar = c(
  "#1B3A7A",  # metro cobalt (negative pole)
  "#5B8DB8",  # GOST pipe blue
  "#E8DFC8",  # institutional cream (neutral)
  "#C45830",  # red lead primer
  "#CD2500"   # Soviet red (positive pole)
)
```

### 4.5 White-Background Usage Guidance (internal metadata)

    dark_only  = c("fuel"),             # brown — fine as fill, merges with black at small sizes
    full_ok    = TRUE,                  # palette is usable in full for fill aesthetics
    lines_subset = c("water","fire","air","toxic","alkali","other")  # 6-color subset safe for lines/points
  ),
  soviet_military = list(
    fill_only  = character(0),
    dark_only  = c("brown_6k"),
    full_ok    = TRUE,
    lines_subset = c("green_4bo","turquoise","primer","soviet_red","sand_7k")

  constructivist = list(
    fill_only  = c("yellow"),
    invisible  = c("cream"),            # near-white on white background
    full_ok    = FALSE,                 # cream unusable as data color on white
    lines_subset = c("red","black","yellow","blue"),  # 4-color subset; drop cream
    note = "Use as 4-color palette on white. Cream (#E8DFC8) is historically grounded but invisible on white backgrounds; it is preserved in the full palette definition as a record of the original design context."
  ),
  gost_signal = list(
    fill_only  = c("caution"),          # yellow — fill only
    full_ok    = TRUE,
    lines_subset = c("danger","safe","mandatory")

  )

## 5. Main Function (`R/sovpal.R`)

### 5.1 `sovpal()` — palette retrieval

```r
sovpal(name, n = NULL, type = NULL, direction = 1, from = 1, to = NULL, subset = NULL)
```

**Arguments:**

- `name` (character): Palette name. Must match a key in `.sovpal_palettes`. Case-insensitive.
- `n` (integer or NULL): Colors to return. If NULL, returns all defined stops. If `n` exceeds stops, interpolation is applied automatically.
- `type` (character or NULL): `"discrete"` or `"continuous"`. If NULL, inferred: discrete if `n` <= stops, continuous otherwise.
- `direction` (integer): `1` (normal) or `-1` (reversed). Applied after all other operations.
- `from` (integer): First stop index to include. Default 1. Use `from = 2` to skip a near-white light anchor on white backgrounds. Does not alter hex values — only selects which stops participate in the gradient.
- `to` (integer or NULL): Last stop index to include. Default NULL (all stops). Use with `from` to select a contiguous range of stops.
- `subset` (character vector or NULL): Named subset of colors to return, by color name. Only valid for qualitative palettes. Example: `subset = c("water","fire","air")` returns only those three colors from `gost14202`. Must match names in the palette; throws informative error if names not found.

**Behavior:**

- Validate `name`: throw error listing all available names if not found.
- Apply `from` / `to` first to restrict the stop range.
- Apply `subset` if provided (qualitative only; error if used with sequential/diverging).
- If `type = "discrete"` and `n` > available colors after subsetting: throw informative error.
- If `type = "continuous"` or inferred continuous: interpolate with `grDevices::colorRampPalette()`.
- Apply `direction` last.
- Return named character vector for discrete; unnamed for continuous.

**Key usage examples to verify correct behavior:**

```r
sovpal("gost14202")
# Returns all 8 colors, named

sovpal("gost14202", subset = c("water","fire","air","toxic","alkali","other"))
# Returns 6-color lines-safe subset, named

sovpal("reactor", from = 2)
# Returns stops 2–5, skipping anti-flash white

sovpal("pripyat", n = 100, type = "continuous", from = 2)
# Interpolates 100 colors from stop 2 to stop 5

sovpal("constructivist", subset = c("red","black","yellow","blue"))
# Returns 4-color white-bg subset, omitting cream

sovpal("coldwar", direction = -1)
# Returns reversed: soviet red → cream → metro cobalt

sovpal("gost14202", n = 3)
# Returns first 3 colors: water, fire, air
```

### 5.2 `show_palette()` — single palette preview

```r
show_palette(name, white_notes = TRUE)
```

Displays a horizontal swatch bar using base R `image()`, with color names and hex codes below. Title shows palette name and type. If `white_notes = TRUE` and the palette has entries in `.sovpal_white_notes`, print a brief usage note to the console (not the plot) after rendering.

### 5.3 `show_all_palettes()` — overview

```r
show_all_palettes(type = NULL)
```

Displays all palettes stacked vertically using base R `par(mfrow)` and `image()`. One row per palette, labeled with name and type tag. If `type` is specified, filter to that type only.

### 5.4 `palette_info()` — metadata retrieval

```r
palette_info(name)
```

Returns a list with: `type`, `n_colors` (number of defined stops), `color_names`, `hex_values`, `white_notes` (from `.sovpal_white_notes`), `lines_subset` (if defined). Useful for programmatic inspection without printing a plot.

## 6. ggplot2 Scale Functions (`R/ggplot2-scales.R`)

All functions check for `ggplot2` with `requireNamespace("ggplot2", quietly = TRUE)` and throw a helpful error if absent.

```r
scale_color_sovpal(palette = "gost14202", discrete = TRUE, direction = 1,
                   from = 1, to = NULL, subset = NULL, ...)
scale_colour_sovpal(...)   # alias for scale_color_sovpal
scale_fill_sovpal(palette = "gost14202", discrete = TRUE, direction = 1,
                  from = 1, to = NULL, subset = NULL, ...)
```

All `sovpal()` parameters (`from`, `to`, `subset`, `direction`) are passed through. This is the mechanism by which white-background adjustments are made at the scale level rather than in the palette definition:

```r
# Full palette as fill (yellow works as bar fill):
scale_fill_sovpal("gost14202")

# Lines-safe subset for point/line aesthetics:
scale_color_sovpal("gost14202",
  subset = c("water","fire","air","toxic","alkali","other"))

# Reactor starting from stop 2 (skip anti-flash white):
scale_fill_sovpal("reactor", discrete = FALSE, from = 2)

# Constructivist without cream:
scale_color_sovpal("constructivist",
  subset = c("red","black","yellow","blue"))
```

**Implementation for discrete:**
```r
ggplot2::discrete_scale(
  aesthetics = "colour",
  scale_name = "sovpal",
  palette = function(n) sovpal(palette, n = n, direction = direction,
                               from = from, to = to, subset = subset),
  ...
)
```

**Implementation for continuous:**
```r
ggplot2::scale_color_gradientn(
  colours = sovpal(palette, n = 256, type = "continuous",
                   direction = direction, from = from, to = to),
  ...
)
```

## 14. Design Decisions & Notes for Claude Code

**On historical fidelity:**
- No hex value in `.sovpal_palettes` may be altered. If a color has poor contrast on white, the solution is `subset`, `from`, or `to` — never changing the value.
- The cream color (#E8DFC8) appears in three palettes (constructivist, pripyat, coldwar) intentionally — it is the same physical color (Soviet institutional wall paint / newsprint) appearing in different contexts.
- The anti-flash white (#F2F3F4) in `reactor` stop 1 is a documented historical color (Soviet nuclear bomber undersides). Its near-whiteness is a feature, not a bug — it represents the lightest end of a real physical range.

**On the `subset` approach:**
- The `subset` parameter is the primary mechanism for white-background adaptation of qualitative palettes. It returns the named subset in the order specified by the user, not the order in the palette definition. This gives full control.
- `lines_subset` entries in `.sovpal_white_notes` are *recommendations*, not enforced restrictions. They are exposed via `palette_info()` for user reference.

**Technical requirements:**
- Do not use `{usethis}` unless confirmed available. Write files directly.
- NAMESPACE generated by roxygen2 only — never write manually.
- ggplot2 in `Suggests` only: all calls use `ggplot2::` prefix inside `requireNamespace()` guards.
- `grDevices` and `graphics` in `Imports` (base R, always available).
